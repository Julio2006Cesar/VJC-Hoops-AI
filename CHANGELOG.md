# CHANGELOG.md

# Registro Oficial de Cambios

Todos los cambios importantes realizados en **VJC Hoops AI** deberán documentarse en este archivo.

El objetivo del CHANGELOG es proporcionar un historial claro, ordenado y cronológico de la evolución del proyecto.

Este documento deberá actualizarse antes de cada publicación oficial y después de completar cualquier Sprint importante definido en `ROADMAP.md`.

---

# Formato

Cada versión utilizará la siguiente estructura.

```text
## [Versión]

### Added
Nuevas funcionalidades.

### Changed
Cambios en funcionalidades existentes.

### Improved
Mejoras de rendimiento o experiencia.

### Fixed
Corrección de errores.

### Removed
Funcionalidades eliminadas.

### Security
Cambios relacionados con seguridad.

### Documentation
Actualizaciones de documentación.
```

---

# Versionado

El proyecto seguirá **Semantic Versioning (SemVer)**.

Formato.

```text
MAJOR.MINOR.PATCH
```

Ejemplos.

```text
0.1.0

0.2.0

0.5.4

1.0.0

2.0.0
```

---

## Significado

### MAJOR

Cambios incompatibles con versiones anteriores.

Ejemplo.

Nueva arquitectura.

Cambio completo del motor.

---

### MINOR

Nuevas funcionalidades compatibles.

Ejemplo.

Nuevo Dashboard.

Editor de personajes.

Sistema de logros.

---

### PATCH

Corrección de errores.

Ejemplo.

Optimización.

Bug Fixes.

Pequeñas mejoras.

# Reglas de Actualización

Toda modificación importante deberá reflejarse en este archivo.

Se deberá crear una nueva entrada cuando ocurra cualquiera de las siguientes situaciones.

* Finalización de un Sprint.
* Publicación de una nueva versión.
* Incorporación de una funcionalidad importante.
* Cambio arquitectónico.
* Corrección de errores críticos.
* Actualización de dependencias importantes.
* Modificación significativa de la documentación.

# Objetivo

Este documento representa el historial oficial de VJC Hoops AI.

Su propósito es facilitar el seguimiento del desarrollo del proyecto, mejorar la transparencia y permitir que cualquier desarrollador o Inteligencia Artificial comprenda rápidamente la evolución del software sin necesidad de revisar todo el historial de commits del repositorio.
